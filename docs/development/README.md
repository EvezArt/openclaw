## What's Here

### Task Prioritization Guide

Learn how to:
- Prioritize tasks using the P0-P3 system
- Resolve conflicts when multiple priorities compete
- Handle emergencies and interrupts
- Communicate progress to stakeholders
- Balance stability work vs. new features

### Development Agenda

The master list of:
- Current development priorities
- Ongoing work items
- Planned refactors
- Technical debt items
- Security considerations

## For Contributors

If you're looking to contribute:

1. **Start here:** [CONTRIBUTING.md](../../CONTRIBUTING.md)
2. **Find work:** [DEVELOPMENT_AGENDA.md](../../DEVELOPMENT_AGENDA.md)
3. **Understand priorities:** [Task Prioritization Guide](task-prioritization-guide.md)
4. **Join the discussion:** [Discord](https://discord.gg/qkhbAGHRBT)

## For Maintainers

If you're maintaining OpenClaw:

1. **Review agenda quarterly:** Update priorities based on user feedback
2. **Use prioritization framework:** Apply consistent decision-making
3. **Track metrics:** Monitor MTTR, test coverage, incident frequency
4. **Communicate changes:** Keep community informed via Discord and changelogs

## Updates

This documentation is living and should be updated as:
- New priorities emerge
- Projects are completed
- Team structure changes
- Community feedback arrives

To propose changes:
1. Open a GitHub Discussion for major changes
2. Submit a PR for minor updates
3. Discuss in Discord #development channel
